# Native Parity UI Test Runbook

## Purpose
Run FileMaker-native UI behavior tests that exercise menus, status/toolbar controls, view/mode transitions, and key object interactions in FMWeb IDE.

## Test suite location
- Spec suite: `tests/ui/native-parity/native-parity.spec.mts`
- Scenario suite: `tests/ui/native-parity/scenarios.spec.mts`
- Scenario definitions: `tests/ui/native-parity/scenarios/`
- Command registry: `tests/ui/native-parity/commandRegistry.mts`
- Reusable invariants: `tests/ui/native-parity/assertions.mts`
- Invariant mapping: `tests/ui/native-parity/invariantMap.ts`
- Engine + coverage report writer: `tests/ui/native-parity/engine.mts`
- Parity taxonomy coverage map: `tests/ui/native-parity/parityMap.ts`
- Fixture manifest: `tests/ui/native-parity/fixtures/parity-fixture-solution.synthetic.json`

## Commands
- Headless smoke (CI default):
  - `npm run test:ui`
- Headed local debug:
  - `npm run test:ui:headed`
- Trace-on run:
  - `npm run test:ui:trace`

## Parity execution engine commands
- Full parity runner with regression gates:
  - `npm run parity:ui`
- Report-only run (no gate failures):
  - `npm run parity:ui:report`
- Explicit baseline update:
  - `ALLOW_PARITY_BASELINE_UPDATE=true npm run parity:ui:update-baseline`

## Framework selection behavior
- Primary framework: **Playwright**
- Runner script: `scripts/run-ui-tests.mts`
- If Playwright CLI is unavailable in the environment, the runner:
  - exits successfully (non-blocking in restricted/offline envs)
  - writes `docs/audit/UI_Command_Coverage_Report.md` with skip reason

## CI behavior
- Workflows run `npm run parity:ui`.
- Artifacts uploaded:
  - `test-results/`
  - `playwright-report/`
  - `docs/audit/Parity_UI_*.md`

## Failure diagnostics
On failure, inspect:
1. Console output for failing test + command id.
2. Playwright artifacts under:
   - `test-results/ui-native-parity/`
   - `test-results/ui-native-parity-html/`
3. Coverage output:
   - `docs/audit/UI_Command_Coverage_Report.md`
4. Parity outputs:
   - `docs/audit/Parity_UI_Scorecard.md`
   - `docs/audit/Parity_UI_Failure_Triage.md`
   - `docs/audit/Parity_UI_Backlog_Auto.md`
   - `docs/audit/Parity_UI_Coverage_Map.md`
   - `docs/audit/Parity_UI_Flakes.md`
   - `docs/audit/Parity_UI_Baseline.json`

## Coverage report
Generated by the engine after command traversal:
- tested
- skipped
- not-found
- error
- computed command coverage percentage
- threshold check (80% of discovered-enabled commands)

Parity runner computes:
- Command coverage
- Invariant pass rate
- Scenario pass rate
- Cross-browser pass rate
- Overall parity score (0-100)
- Flake budget and new-flake detection

## Adding a new UI command test
1. Add command to `tests/ui/native-parity/commandRegistry.mts` with:
   - id, label, category, mode(s)
   - selector and/or menu + item text
2. If needed, add a reusable assertion in `assertions.mts`.
3. Extend `native-parity.spec.mts` to call the command directly or rely on harness traversal.
4. Run `npm run test:ui` and verify `UI_Command_Coverage_Report.md` updates.

## Adding a new fixture
1. Add fixture metadata under `tests/ui/native-parity/fixtures/`.
2. Prefer real DDR-derived fixture layouts.
3. If synthetic, label clearly as `SYNTHETIC` and document limitations.
4. Update:
   - `docs/audit/UI_Native_Behavior_Test_Catalog.md`
   - `docs/audit/Edge_Case_Playbook_FileMaker_on_Web.md`

## Troubleshooting checklist
- App not loading in tests:
  - verify dev server command in `playwright.ui.config.mts`
  - verify route `/layouts/<layout>/browse?workspace=default&uiTest=1`
- Menu item not found:
  - confirm menu text and aria labels in `components/layout-mode.tsx` / `components/browse-mode.tsx`
- Flaky focus assertion:
  - ensure selectors target visible editable controls only
  - avoid fixed sleeps; use explicit state assertions
- Coverage threshold failure:
  - inspect report for many `not-found`/`error` commands
  - update registry selectors or preconditions
